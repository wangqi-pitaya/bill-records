## [0.1.0] - 2026-07-03

### 新增功能

#### 首页账单列表
- 实现账单列表展示，支持按日期分组（同一天归为一组）
- 日期卡片显示当天收支汇总（支出红色/收入绿色）
- 顶部标题栏固定，点击日期区域弹出日期选择弹层
- 日期选择弹层支持按年/按月两种模式切换：
  - 按年模式：仅显示年份列表，筛选整年账单
  - 按月模式：显示年份 + 月份列表，筛选指定年月账单
- 日期选择弹层从顶部向下弹出，无标题栏，通过背景点击或底部按钮关闭
- 日期选择弹层打开时禁用背景页面滚动
- 年份自动跳转：无数据时自动切换到最近有数据的年份
- 统计卡片展示收入、支出、余额汇总
- 悬浮新增按钮（+），滚动时自动显隐
- 账单项左滑显示编辑/删除按钮
- 列表滚动时自动收起已展开的左滑操作

#### 新建/编辑账单抽屉
- 底部全屏抽屉式新建页面，URL 参数控制状态
- 支出/收入 Tab 切换，分类列表联动更新
- 分类网格选择（5列布局），默认选中第一个
- 分类选中高亮（支出红色、收入绿色），未选中白色底色
- 正方形分类按钮，图标大、留白小
- 备注输入框，铺满剩余空间
- 金额输入框，支持完整金额校验：
  - 禁止多个小数点
  - 禁止小数点开头（自动补0）
  - 多个0开头自动合并
  - 最多两位小数
  - 最小宽度父元素一半，最大宽度80%
- 日期选择弹窗，支持今天/昨天/前天快捷选择
- 日历组件支持月份切换和日期点选
- "再记"按钮支持连续新建
- "保存"按钮保存并返回
- 编辑功能：预填充数据，保存后关闭抽屉
- 返回按钮关闭抽屉，手机返回键适配

#### 账单操作
- 左滑手势：支持触摸滑动和鼠标拖拽
- 编辑功能：点击编辑按钮打开抽屉预填充数据
- 删除功能：二次确认弹窗，防止误删
- 保存后返回列表不滚动到顶部

#### 分类管理
- 独立分类管理页面 `/categories`
- 支出/收入 Tab 切换
- 分类网格展示（5列），与新建页风格一致
- 每个分类右上角删除按钮（减号图标）
- 删除保护：已有账单的分类禁止删除并提示
- 添加分类弹窗：名称输入 + 32个图标选择
- 拖拽排序：桌面端 HTML5 拖拽 + 移动端触摸拖拽
- 返回按钮回到新建抽屉

#### 数据与存储
- localStorage 本地持久化存储
- Zustand 状态管理
- 首次使用自动初始化默认分类和示例账单
- 默认8个支出分类、5个收入分类
- 7条示例账单数据

#### 交互体验
- 抽屉打开/关闭动画过渡
- 左滑操作动画反馈
- 按钮悬停和点击状态反馈
- 空状态友好提示
- 确认弹窗统一样式
- 响应式布局，移动端优先

### 技术实现

#### 技术栈
- React 18 + TypeScript
- Vite 构建工具
- Tailwind CSS 样式方案
- React Router v6 路由管理
- Zustand 状态管理
- Lucide React 图标库

#### 核心组件
- `Home` - 首页容器
- `AddBillDrawer` - 新建/编辑抽屉
- `CategoryManage` - 分类管理页
- `BillItem` - 账单列表项（支持左滑）
- `CategoryGrid` - 分类网格
- `StatCard` - 统计卡片
- `DatePicker` - 日期选择弹窗
- `ConfirmModal` - 确认弹窗
- `FloatingButton` - 悬浮按钮

#### 状态管理
- `useBillStore` - 账单数据（增删改查 + 统计）
- `useCategoryStore` - 分类数据（增删改 + 排序 + 使用检测）

### 修复问题

- 抽屉状态使用 URL 参数管理，修复浏览器返回按钮行为
- 修复从分类管理返回时抽屉抖动问题
- 修复 TypeScript 类型错误（lucide-react 图标导入）
- 优化左滑手势与页面滚动的冲突处理
- 优化金额输入框唤起键盘的灵敏度
- 修复日期选择弹层和新建账单抽屉打开时背景页面滚动的问题

### 已知限制

- 数据仅保存在当前浏览器 localStorage 中，无法跨设备同步
- 不支持数据导入/导出功能
- 不支持多币种
- 不支持账单搜索功能
- 统计仅支持按年月筛选，不支持自定义时间段
